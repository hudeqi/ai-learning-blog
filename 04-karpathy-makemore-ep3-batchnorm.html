1. 这一集解决什么问题

Ep.2 我们已经搭起了一个 MLP，能把 dev loss 降到 ~2.17。但是有几个奇怪的现象：

  1. 训练一开始 loss 是 ~27，而理论值应该是 -log(1/27) ≈ 3.30 才对
  2. 把激活值打印出来，发现 tanh 输出大量贴在 ±1（"饱和"）
  3. 训练曲线开头很平，模型"前 1000 步几乎不学习"

这一集 Karpathy 不是来教你新的模型结构 —— 他用 两个具体的工具（Kaiming 初始化、BatchNorm）+ 一套诊断方法，让你看清"神经网络为什么能训练 / 为什么训练不动"的底层机制。

这一集的核心思想（一句话）：
神经网络的每一层都希望"输入的激活分布是均值 0、标准差 1"。 只要这个条件被破坏，训练就会出问题。 整集就是围绕"如何维持这个条件"展开。

2. 第一个症状：初始 loss 远大于理论值

2.1 理论值是多少

vocab_size = 27，刚初始化的模型应该对每个字符给均匀概率 1/27。NLL = -log(1/27) ≈ 3.30。

2.2 实测是多少

跑一下 Ep.2 的初始化代码：

logits = h @ W2 + b2     # (32, 27)
loss = F.cross_entropy(logits, Y)
print(loss.item())       # ≈ 27 ！！比 3.30 大 10 倍

2.3 原因：W2 / b2 太大，logits 分布太宽

初始时 W2 = randn(...)、b2 = randn(27)，logits 中有些值偏大（比如 +5）、有些值偏小（比如 -5），softmax 后某些位置接近 1，某些位置接近 0。

如果真实标签恰好是"接近 0 的那个位置"，-log(接近 0 的概率) = 很大的数字，loss 就爆了。

2.4 修复：让 logits 一开始更"平"

W2 = torch.randn((n_hidden, vocab_size)) * 0.01   # 缩小到 1/100
b2 = torch.zeros(vocab_size)                       # 干脆置 0

这样初始 logits 几乎全是 0，softmax 后接近均匀分布，初始 loss ≈ 3.30 ✓

💡 为什么不直接 W2 = zeros
全 0 的 W2 会让所有 logits 完全相等，反向传播时所有梯度都一样，所有神经元学到的东西完全相同（symmetry breaking 失败）。乘 0.01 既小又有差异，是最优解。

3. 第二个症状：tanh 大量饱和（"白条图"）

3.1 现象

把隐藏层激活 h = tanh(hpreact) 打印成图：

plt.figure(figsize=(20, 10))
plt.imshow((h.abs() > 0.99).float(), cmap='gray', interpolation='nearest')

会看到一张"布满白色横条"的图：

神经元编号 → 0 1 2 3 4 ... 99 样本 0 : ■ □ □ ■ □ □ □ ■ □ ■ 样本 1 : ■ □ □ ■ □ □ □ ■ □ ■ ← 神经元 0、3、7、9 样本 2 : ■ □ □ ■ □ □ □ ■ □ ■ 对所有样本都饱和 → "死神经元" 样本 31 : ■ □ □ ■ □ □ □ ■ □ ■

整列整列地饱和意味着这些神经元的输出永远是 ±1，梯度永远 ≈ 0，训练时它们的参数完全得不到更新。

3.2 一个反向思考的疑问

🤔 tanh 在 ±∞ 时饱和，那 hpreact 不该集中在 ±∞ 才会饱和吗？

这是个非常容易混淆的点。tanh 的形状（饱和在 ±1）和它输入的分布是两件独立的事：

  • tanh 的形状是函数定义本身，恒定
  • hpreact 的分布由前一层的初始化和数据决定，几乎总是近似高斯

高斯分布天然集中在 0 附近，但 std 太大时仍然会有 20-30% 的值溢出到饱和区。这才是问题所在。

3.3 为什么 hpreact 会"过大"

初始时 W1 = randn((6, 100))，每个元素 std ≈ 1。计算：

hpreact[i, j] = emb[i,0]*W1[0,j] + emb[i,1]*W1[1,j] + ... + emb[i,5]*W1[5,j] + b1[j]
              ↑           6 项独立、零均值的乘积相加              ↑

统计学常识：Var(X*Y) = Var(X) * Var(Y)，Var(X+Y) = Var(X) + Var(Y)（独立时）。

量方差
emb[i,k]（标准正态）1
W1[k,j]（标准正态）1
乘积1×1 = 1
6 项求和1+1+1+1+1+1 = 6

所以 hpreact.std() ≈ √6 ≈ 2.45 —— 这个 std 就是导致 tanh 大面积饱和的元凶。

4. 权重初始化的完整推导（Kaiming）

这是视频 30-40min 那段，是整集最关键也最容易绕晕的 10 分钟。下面拆成 4 步给你看。

Step 1：目标

让每一层激活的 std 保持在 1 附近，既不饱和也不萎缩。

Step 2：解决"6 项求和让方差变 6"的问题

既然 6 个项相加让方差从 1 涨到 6，那就缩小 W，让每个项的方差从 1 缩到 1/6：

$$ W = \mathcal{N}(0, 1) \times c, \quad \text{使得 } 6c^2 = 1 \implies c = \frac{1}{\sqrt{6}} $$

一般化为 Xavier 初始化：

$$ \boxed{\;W \sim \mathcal{N}(0, 1) \times \frac{1}{\sqrt{\text{fan\_in}}}\;} $$

这里 fan_in = 该层输入维度（W shape 是 (fan_in, fan_out) 的话，就是第一个数）。

Step 3：但是 tanh 会"压扁"std

把一个 std=1 的高斯过 tanh，输出的 std 不是 1，而是约 0.63（tanh 在两端饱和把数据压扁了）。如果每层都被压一次：

第 1 层 std = 0.63 第 2 层 std = 0.63² = 0.40 第 3 层 std = 0.63³ = 0.25 ... 第 10 层 std ≈ 0.01 ← 信号几乎消失

Step 4：用 gain 补偿

事先把 W 放大 1/0.63 ≈ 1.59 倍，过完 tanh 正好恢复 std=1。这个补偿系数叫 gain。

激活函数gain
linear / 恒等1.0
tanh5/3 ≈ 1.667
ReLU√2 ≈ 1.414

最终公式（Kaiming 初始化）

$$ \boxed{\;W \sim \mathcal{N}(0, 1) \times \frac{\text{gain}}{\sqrt{\text{fan\_in}}}\;} $$

fan_in = block_size * n_embd        # 例如 30
W1 = torch.randn((fan_in, n_hidden)) * (5/3) / fan_in**0.5
b1 = torch.zeros(n_hidden)           # bias 用小值或 0

验证：跑一下三种初始化

import torch, math

emb = torch.randn(1000, 6)

for label, W_scale in [
    ('朴素初始化',         1.0),
    ('Xavier (除 √fan_in)', 1.0 / math.sqrt(6)),
    ('Kaiming (带 gain=5/3)', (5/3) / math.sqrt(6)),
]:
    W = torch.randn(6, 100) * W_scale
    h = torch.tanh(emb @ W)
    print(f'{label:25s}  hpreact std={(emb @ W).std():.3f}  tanh std={h.std():.3f}  '
          f'饱和={((h.abs()>0.99).float().mean()):.1%}')

预期输出（约略）：

朴素初始化                    hpreact std=2.450  tanh std=0.768  饱和=18.0%
Xavier (除 √fan_in)         hpreact std=1.000  tanh std=0.626  饱和=0.3%
Kaiming (带 gain=5/3)       hpreact std=1.668  tanh std=0.928  饱和=3.5%
关键洞察：Kaiming 的 hpreact std=1.67 看起来"比 Xavier 的 1.0 还大"，但过完 tanh 之后反而是 Kaiming 的 std 接近 1（最健康）。 真正要稳定的是激活函数的输出，不是它的输入。

5. BatchNorm：强制把激活拉回正常区间

Kaiming 初始化有 3 个局限：

  1. 只对初始化的那一刻有效。训练几步后 W 被更新，理论保证失效
  2. 深网络误差累积。第 10 层 std 可能漂到 0.7 或 1.5
  3. 对每种激活函数都要重新算 gain。GELU、SwiGLU 没有干净的解析解

BatchNorm 的想法：与其精心设计初始化，不如在 forward 里强行把 hpreact 标准化。

5.1 BatchNorm 的 4 步

# Step 1：算 batch 的均值（每个神经元独立）
bnmean = hpreact.mean(0, keepdim=True)         # shape (1, 200)

# Step 2：算 batch 的标准差
bnstd  = hpreact.std(0, keepdim=True)          # shape (1, 200)

# Step 3：标准化
hpreact_norm = (hpreact - bnmean) / bnstd      # mean=0, std=1

# Step 4：可学习的缩放和平移
hpreact = bngain * hpreact_norm + bnbias       # γβ

5.2 为什么必须沿 dim=0（样本维度）

hpreact shape (32, 200) = 32 个样本 × 200 个神经元。

  • mean(0) 跨样本：每个神经元独立标准化 ✅
  • mean(1) 跨神经元：把 200 个不同神经元混起来 ❌（语义错误）

每个神经元有自己的"语义"（一个学元音、一个学辅音...），不能混着标准化。

5.3 为什么要 γβ 这两个可学习参数

这是 BatchNorm 设计中最深的一招

强制 std=1, mean=0 是不是过度约束？也许某层的最佳分布是 std=2 或 mean=0.3？

γβ 让模型自己说了算：BN 给了一个良好起点，γβ 让模型自由微调。

初始：bngain=1, bnbias=0 → 等于纯标准化
训练后：bngain 和 bnbias 收敛到最适合这层的值

5.4 一个常见担忧

🤔 既然 γβ 自由调，模型不会学到 bngain=5 让 tanh 重新饱和吗？

理论上可能，但实际不严重，因为有自我修正机制：

  • bngain 大 → tanh 饱和 → 梯度消失 → 推 bngain 变小
  • bngain 小 → tanh 退化为线性 → loss 变差 → 推 bngain 变大

梯度下降会自动找到 trade-off。实测多数神经元的 bngain 收敛到 0.3~1.8 区间，极少越界。

6. 训练 vs 推理：BN 必须区分的两种模式

6.1 训练时一切正常，但推理时呢？

BN 在训练时用 当前 batch 的 mean/std。但推理时往往只有 1 个样本：

hpreact.shape   # (1, 200)
hpreact.std(0)  # 全是 0 ！每列只有一个数，std 不可算

→ 推理时不能用 batch 统计量。

6.2 解决方案：训练时维护 running 统计量

bnmean_running = torch.zeros((1, n_hidden))
bnstd_running  = torch.ones((1, n_hidden))

# 训练时每步用 EMA 更新
with torch.no_grad():
    bnmean_running = 0.999 * bnmean_running + 0.001 * bnmean_i
    bnstd_running  = 0.999 * bnstd_running  + 0.001 * bnstd_i

# 推理时用 running 统计量
hpreact_norm = (hpreact - bnmean_running) / bnstd_running

6.3 关于那个 0.001（EMA / 指数移动平均）

通用公式 new = (1-α) * old + α * current。α 越小，平滑越强。

α有效平均窗口适用场景
0.1（PyTorch 默认）最近 10 个 batch训练步数较少（<1万）
0.01最近 100 个 batch一般场景
0.001（Karpathy 选择）最近 1000 个 batch20 万步训练 / LLM 预训练
⚙️ Infra 视角
EMA 在 ML 系统里到处都是：Adam 优化器、loss 监控曲线、StyleGAN/Diffusion 的 EMA model、训练监控平滑。 对 Kafka 内核工程师来说，这就是 broker 监控里的 EWMA（指数加权移动平均），思想完全一样。

6.4 PyTorch 标准做法

# 训练时
model.train()
output = model(X_batch)
loss.backward()

# 推理 / 评估时
model.eval()                 # ← BN 自动切换到用 running 统计量
with torch.no_grad():        # ← 关闭梯度跟踪，省显存
    output = model(X_new)
经典 bug：推理时忘了 model.eval()，BN 用 1 个样本的统计量（std=0），结果错乱。面试常考。

7. 诊断工具：3 张图诊断网络健康

Karpathy 视频里反复用这 3 张图诊断模型，是真实工程中的"训练 debug 工具"。

7.1 激活分布直方图

plt.hist(h.flatten().tolist(), 50)

健康模型应该看到一个钟形分布，集中在 0 附近，两端衰减。如果两端有"尖峰"（贴在 ±1），说明饱和。

7.2 梯度分布直方图

plt.hist(h.grad.flatten().tolist(), 50)

梯度也应该是钟形。如果梯度集中在 0，说明梯度消失。

7.3 参数:梯度比（每个参数）

for p in parameters:
    ratio = (p.grad.std() / p.data.std()).log10().item()
    print(f'shape {tuple(p.shape):20s}  ratio = {ratio:.3f}')

经验法则：这个 ratio 应该在 -3 左右（即梯度的 std 约为参数的 1/1000）。

  • ratio > -1：梯度太大，可能 loss 爆炸
  • ratio < -5：梯度太小，参数几乎不更新

8. PyTorch nn.Module 风格重构

视频最后把代码重写成 PyTorch 的标准模块化风格。这一段没有新概念，但是从这一集开始所有代码都用这种风格，要适应。

class Linear:
    def __init__(self, fan_in, fan_out, bias=True):
        self.weight = torch.randn((fan_in, fan_out)) / fan_in**0.5
        self.bias = torch.zeros(fan_out) if bias else None
    def __call__(self, x):
        self.out = x @ self.weight
        if self.bias is not None:
            self.out += self.bias
        return self.out
    def parameters(self):
        return [self.weight] + ([] if self.bias is None else [self.bias])


class BatchNorm1d:
    def __init__(self, dim, eps=1e-5, momentum=0.1):
        self.eps = eps
        self.momentum = momentum
        self.training = True
        # 可学习参数
        self.gamma = torch.ones(dim)
        self.beta = torch.zeros(dim)
        # 缓冲区（不参与训练，只随训练数据更新）
        self.running_mean = torch.zeros(dim)
        self.running_var = torch.ones(dim)
    def __call__(self, x):
        if self.training:
            xmean = x.mean(0, keepdim=True)
            xvar = x.var(0, keepdim=True)
        else:
            xmean = self.running_mean
            xvar = self.running_var
        xhat = (x - xmean) / torch.sqrt(xvar + self.eps)
        self.out = self.gamma * xhat + self.beta
        # 更新 running 统计量
        if self.training:
            with torch.no_grad():
                self.running_mean = (1 - self.momentum) * self.running_mean + self.momentum * xmean
                self.running_var = (1 - self.momentum) * self.running_var + self.momentum * xvar
        return self.out
    def parameters(self):
        return [self.gamma, self.beta]


class Tanh:
    def __call__(self, x):
        self.out = torch.tanh(x)
        return self.out
    def parameters(self):
        return []

组装成一个网络：

n_embd, n_hidden = 10, 100
C = torch.randn((vocab_size, n_embd))
layers = [
    Linear(n_embd * block_size, n_hidden, bias=False),  # bias 被 BN 接管，可省
    BatchNorm1d(n_hidden),
    Tanh(),
    Linear(n_hidden, n_hidden, bias=False),
    BatchNorm1d(n_hidden),
    Tanh(),
    # ... 可以堆 6 层 ...
    Linear(n_hidden, vocab_size, bias=False),
    BatchNorm1d(vocab_size),                            # 最后一层也 BN
]
💡 BN 后 bias 是冗余的
Linear(..., bias=True) 加的 bias 会被紧跟其后的 BN 的 减 bnmean 抵消，效果由 bnbias 接管。 所以 Linear + BatchNorm 的标准写法是 Linear(..., bias=False)。

9. Infra 视角：这一集真正埋下的工程伏笔

🛠 这一集对 AI Infra 转型的核心价值
  1. 初始化敏感性 = 大模型训练 loss spike 的根因之一
    千亿模型预训练时，loss 偶尔会突然飙升（spike）然后崩掉。一个主要原因就是深层激活的 std 漂移到了危险区间。Megatron-LM 有专门的初始化策略来缓解。
  2. BatchNorm → LayerNorm/RMSNorm 的演化
    LLM 不用 BatchNorm，用 LayerNorm（每个样本独立标准化，不依赖 batch 维度）。但"强制标准化 + γβ 可学习"的思想完全一致。 RMSNorm 是 LayerNorm 的简化版（只算 std，不算 mean），LLaMA 系列在用，省 20% 计算。
  3. 训练 / 推理模式切换是推理引擎的核心常识
    vLLM 推理时全程在 model.eval() + torch.no_grad() 模式，BN/LN/Dropout 都用 running 统计量或关闭。 忘记切换是经典 bug，面试官常问。
  4. EMA 思想在训练系统里到处都是
    Adam 优化器的 betas=(0.9, 0.999) 就是 EMA 的 α；loss 平滑显示用 EMA； StyleGAN/Diffusion 模型的"EMA weights"也是这个思路。对 Kafka 内核工程师来说，这就是 broker 监控里的 EWMA。
  5. 诊断工具是面试 deep talk 的核武器
    "训练不稳定怎么 debug" → 看激活分布、看梯度分布、看参数:梯度比。 这一集教的 3 张图就是工业界做训练 debug 的标准动作。
  6. BN 的 batch size 依赖 → 现代 LLM 抛弃它的原因
    序列任务里"一个 batch 内不同样本序列长度不同"导致 BN 统计不稳。 LayerNorm 解决了这个问题——它在每个 token 内独立标准化，与 batch 维度无关。 这是 Transformer 论文选 LayerNorm 而非 BN 的根本原因。

10. 可直接运行的完整代码

"""
makemore Ep.3 —— Kaiming 初始化 + BatchNorm
"""
import torch
import torch.nn.functional as F
import random
import math

# ============ 1. 读数据 + 字符表 ============
words = open('names.txt').read().splitlines()
chars = sorted(set(''.join(words)))
stoi = {s: i + 1 for i, s in enumerate(chars)}
stoi['.'] = 0
itos = {i: s for s, i in stoi.items()}
V = len(stoi)

# ============ 2. 数据集 ============
block_size = 3
def build_dataset(ws):
    X, Y = [], []
    for w in ws:
        ctx = [0] * block_size
        for ch in w + '.':
            ix = stoi[ch]
            X.append(ctx)
            Y.append(ix)
            ctx = ctx[1:] + [ix]
    return torch.tensor(X), torch.tensor(Y)

random.seed(42); random.shuffle(words)
n1, n2 = int(0.8 * len(words)), int(0.9 * len(words))
Xtr,  Ytr  = build_dataset(words[:n1])
Xdev, Ydev = build_dataset(words[n1:n2])
Xte,  Yte  = build_dataset(words[n2:])

# ============ 3. 模型参数（Kaiming + BN） ============
n_embd, n_hidden = 10, 200
g = torch.Generator().manual_seed(2147483647)

C  = torch.randn((V, n_embd), generator=g)

# W1：Kaiming 初始化（带 tanh gain）
W1 = torch.randn((block_size * n_embd, n_hidden), generator=g) * (5/3) / (block_size * n_embd)**0.5
# 不需要 b1（被 BN 接管）

# W2：缩小到 0.01，初始 logits 接近 0
W2 = torch.randn((n_hidden, V), generator=g) * 0.01
b2 = torch.zeros(V)

# BatchNorm 参数
bngain = torch.ones((1, n_hidden))
bnbias = torch.zeros((1, n_hidden))

# 推理时用的 running 统计量（不参与训练）
bnmean_running = torch.zeros((1, n_hidden))
bnstd_running  = torch.ones((1, n_hidden))

parameters = [C, W1, W2, b2, bngain, bnbias]
for p in parameters:
    p.requires_grad = True
print(f'#params = {sum(p.nelement() for p in parameters)}')

# ============ 4. 训练循环 ============
batch_size = 32
max_steps = 200_000

for step in range(max_steps):
    ix = torch.randint(0, Xtr.shape[0], (batch_size,))
    Xb, Yb = Xtr[ix], Ytr[ix]
    
    # === forward ===
    emb = C[Xb]
    embcat = emb.view(emb.shape[0], -1)
    hpreact = embcat @ W1                          # (32, 200)
    
    # --- BatchNorm ---
    bnmeani = hpreact.mean(0, keepdim=True)
    bnstdi  = hpreact.std(0, keepdim=True)
    hpreact = bngain * (hpreact - bnmeani) / bnstdi + bnbias
    
    # 更新 running 统计量
    with torch.no_grad():
        bnmean_running = 0.999 * bnmean_running + 0.001 * bnmeani
        bnstd_running  = 0.999 * bnstd_running  + 0.001 * bnstdi
    
    h = torch.tanh(hpreact)
    logits = h @ W2 + b2
    loss = F.cross_entropy(logits, Yb)
    
    # === backward ===
    for p in parameters:
        p.grad = None
    loss.backward()
    
    # === update ===
    lr = 0.1 if step < 100_000 else 0.01
    for p in parameters:
        p.data += -lr * p.grad
    
    if step % 10_000 == 0:
        print(f'step {step:6d}  loss = {loss.item():.4f}')

# ============ 5. 评估（推理模式） ============
@torch.no_grad()
def split_loss(split):
    X, Y = {'train': (Xtr, Ytr), 'dev': (Xdev, Ydev), 'test': (Xte, Yte)}[split]
    emb = C[X]
    embcat = emb.view(emb.shape[0], -1)
    hpreact = embcat @ W1
    # 推理模式：用 running 统计量
    hpreact = bngain * (hpreact - bnmean_running) / bnstd_running + bnbias
    h = torch.tanh(hpreact)
    logits = h @ W2 + b2
    loss = F.cross_entropy(logits, Y)
    print(f'{split} loss = {loss.item():.4f}')

split_loss('train')
split_loss('dev')

# ============ 6. 采样 ============
g = torch.Generator().manual_seed(2147483647 + 10)
for _ in range(20):
    out, ctx = [], [0] * block_size
    while True:
        emb = C[torch.tensor([ctx])]
        embcat = emb.view(1, -1)
        hpreact = embcat @ W1
        hpreact = bngain * (hpreact - bnmean_running) / bnstd_running + bnbias
        h = torch.tanh(hpreact)
        logits = h @ W2 + b2
        probs = F.softmax(logits, dim=1)
        ix = torch.multinomial(probs, 1, generator=g).item()
        ctx = ctx[1:] + [ix]
        if ix == 0:
            break
        out.append(itos[ix])
    print(''.join(out))

  • 初始 loss ≈ 3.30（不再是 ~27）
  • 训练前期 loss 立即下降（不再有"卡 1000 步"）
  • "白条图"几乎全黑（饱和神经元 < 5%）

11. ⭐ 学习过程中遇到的高频卡点答疑（10 个 Q）

这一集是整个系列里"信息密度最高"的一集，下面是我在学这一集时实际卡住的所有问题。按视频时间顺序整理。

Q1：@torch.no_grad() 装饰器是什么意思？不算梯度怎么训练？

一句话：@torch.no_grad() 用在"评估"函数上，不用在"训练循环"上。训练循环里仍然正常算梯度、反向传播、更新参数。

原理：PyTorch 默认在每个张量运算时偷偷建一张"反向求导图"（autograd graph），这要花显存和算力。torch.no_grad() 让接下来的运算不建图、不存中间值。

什么时候用：

场景加 no_grad 吗？
训练（forward + backward + update）❌ 不加
计算 dev/test loss✅ 加
推理 / 生成样本✅ 加
梯度统计、画激活直方图✅ 加
Q2：hpreact 是什么？读 "react" 还是 "pre-act"？

读 "h pre-act"，是 hidden pre-activation 的缩写 —— 隐藏层在过激活函数之前的值。

hpreact = emb_cat @ W1 + b1     # 隐藏层"激活前"的值
h       = torch.tanh(hpreact)   # 隐藏层"激活后"的值

为什么要分开命名：诊断激活函数的"健康程度"必须看它的输入（pre-activation），因为输出已经被压扁过了，看不出原始信号有多大。

术语家族（指同一类东西）：pre-activation、logits（特指最后一层）、linear output、数学符号 z。

Q3：tanh 在 ±∞ 时饱和，hpreact 应该集中在 ±∞ 才会饱和吗？为什么打印出来分布在 0 附近？

你混淆了"因果关系"。tanh 的形状（饱和在 ±1）和它输入的分布是两件独立的事。

  • tanh 的形状是函数定义本身，恒定
  • hpreact 的分布由前一层的初始化和数据决定，几乎总是近似高斯

关键：hpreact 集中在 0 附近不代表 tanh 不饱和。高斯分布的"集中"是相对的——std=2.5 时，仍有 ~30% 的值落到 ±3 之外，这部分就在 tanh 饱和区。

所以问题应该问"为什么前一层会输出 std=2.5 这么大的分布"，答案在 Q5。

Q4：std=0.1（太小）为什么也不行？输入小不是更安全吗？

因为 tanh 在 |x|<0.5 时近似线性，整个深网络坍缩成单层线性变换：

当 hpreact 全在 ±0.1：
h = tanh(hpreact) ≈ hpreact           # tanh 退化为恒等
out = h @ W2 + b2 ≈ hpreact @ W2 + b2
    = X @ (W1 @ W2) + (b1 @ W2 + b2)  # 等价于一层线性！

同时梯度也会逐层缩小（信号 0.1 → 0.01 → 0.001 → ...），梯度消失。

结论：std 太大和太小都不行。两者都让信号传不动，只是机制不同：

  • 太大 → tanh 饱和 → 梯度被掐断
  • 太小 → tanh 退化为线性 → 信号萎缩，深网络等于一层
Q5：30-40min 那段权重初始化推导没听懂，能再讲一遍吗？

4 步走：

  1. 问题：朴素初始化 W=randn(6,100) 会让 hpreact std ≈ √6 ≈ 2.45（因为 6 项独立项相加，方差加和）
  2. 缩小 W：让 W = randn * 1/√fan_in，则 hpreact std=1（这就是 Xavier）
  3. 但 tanh 会压扁 std：输入 std=1 过完 tanh 变 0.63，深层信号衰减
  4. 用 gain 补偿：W = randn * gain/√fan_in，tanh 的 gain=5/3，过完 tanh 正好恢复 std=1

最终公式（Kaiming/He 初始化）：

$$ W \sim \mathcal{N}(0, 1) \times \frac{\text{gain}}{\sqrt{\text{fan\_in}}} $$

fan_in 是该层输入维度；gain 是激活函数的修正系数（tanh: 5/3；ReLU: √2；linear: 1）。

Q6：既然 Kaiming 能搞定，为什么还要 BatchNorm？不是每层都能用 Kaiming 干预吗？

Kaiming 是一次性的"施工"，BN 是每个 forward 都重新"维护"：

对比Kaiming initBatchNorm
何时生效仅初始化时每次 forward
训练中漂移会，无法修正自动重新校准
深网络误差累积每层独立保证
激活函数依赖强（要算 gain）几乎无关
开销0（一次性）每次 forward 都要算

"BN 就是持续生效的、不依赖理论假设的、自动版的 Kaiming"。

Q7：BN 后又乘 γβ "撤销"标准化，那 tanh 不会又饱和吗？

理论上可能，但实际不严重，原因是梯度下降的自我修正机制：

  • bngain 太大 → tanh 饱和 → 梯度 ≈ 0 → 推 bngain 变小
  • bngain 太小 → tanh 退化为线性 → loss 变差 → 推 bngain 变大

梯度下降会自动找到平衡点。实测多数神经元的 bngain 收敛到 0.3~1.8 区间，极少越界。

初始时 bngain=1, bnbias=0，BN 等于纯标准化——这是个良好起点，模型从这里渐进微调，不会突然跳到糟糕位置。

Q8："训练"和"推理"特指什么？
模式目的包含步骤
训练调参forward + backward + 更新参数
推理用，不调仅 forward，不更新参数

实际场景：

  • 训练循环跑 200k 步 → 训练
  • 训练中每 10k 步评估一次 dev loss → 推理（不更新参数）
  • 训练完后采样生成名字 → 推理
  • 用户用 ChatGPT 问答 → 推理

必须区分的原因：BatchNorm（用 batch 还是 running 统计量）、Dropout（保留还是丢弃）、梯度跟踪（开还是关）在两种模式下行为不同。

Q9：bnmean_running = 0.999 * old + 0.001 * current 中的 0.999 / 0.001 怎么来的？

这是 EMA（指数移动平均），通用公式 new = (1-α) * old + α * current。α=0.001 意味着"每次只吸收 0.1% 新值，保留 99.9% 历史"。

等价于"对最近 ~1/α 个 batch 做平均"：

α有效平均窗口
0.1（PyTorch 默认）最近 10 个 batch
0.01最近 100 个
0.001（Karpathy 选）最近 1000 个

为什么不直接用当前值：单 batch 统计量噪声大，会让 running 抖动。EMA 平滑掉抖动，反映长期趋势。

为什么 Karpathy 选 0.001：训练 20 万步，0.001 让 running 反映"整个训练的稳态"而不是"最近几个 batch"。

Q10："tanh 唯一意义是引入非线性"对吗？

不严谨。准确说法：

  • 所有激活函数（tanh、ReLU、GELU）的核心作用都是引入非线性 —— 没有它，深网络坍缩成单层
  • tanh 相对其他激活函数还有 3 个特殊性质：输出有界 [-1, 1]、零中心、平滑可导

现代 LLM 已经基本不用 tanh 了，主流是 GELU/SwiGLU。但"激活函数必须引入非线性"这个底层逻辑没变。
