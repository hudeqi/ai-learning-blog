1. makemore 是什么 · 任务定义

makemore 是 Karpathy 在 "Neural Networks: Zero to Hero" 系列中第二个项目。它的目标非常朴素：

给定一份「真实的英文名字」清单（约 32,000 个），训练一个模型， 让它能生成「看起来像名字」的新字符串，例如： emma, olivia, ava, ... → jazlyn, kaelin, novahlee, ...

这本质上就是一个 字符级语言模型 (Character-Level Language Model)。它和 GPT 的目标完全一致：

  • GPT：给定前面若干 token，预测下一个 token
  • makemore：给定前面若干字符，预测下一个字符

第 1 集采用最简单的形式 —— Bigram 模型：只用前一个字符来预测下一个字符。 也就是说，模型的"记忆"窗口长度 = 1。

这是个非常弱的模型，但它麻雀虽小五脏俱全：建模、训练、采样、评估、损失函数、参数更新… 所有语言模型的"骨架"它都有。后面 4 集会一步步把这个骨架升级成 MLP → WaveNet → Transformer。

2. 数据加载 & 字符表 & 特殊 token

2.1 原始数据

words = open('names.txt', 'r').read().splitlines()
# ['emma', 'olivia', 'ava', 'isabella', 'sophia', ...]
len(words)  # 32033

2.2 构建字符表

把所有出现过的字符收集起来，并加入一个特殊 token '.' 来同时表示 "序列开始" 和 "序列结束"。

chars = sorted(list(set(''.join(words))))   # ['a', 'b', ..., 'z']
stoi = {s: i+1 for i, s in enumerate(chars)} # a→1, b→2, ..., z→26
stoi['.'] = 0                                # 特殊 token '.' → 0
itos = {i: s for s, i in stoi.items()}       # 反向映射
vocab_size = len(stoi)                       # 27
💡 为什么用一个 token . 同时表示开始 & 结束？
Karpathy 的处理很简洁：每个名字 "emma" 被扩展成 ". e m m a ."， 这样 Bigram 训练时既学到「以 . 开头时，第一个字符的分布」， 也学到「什么字符后面容易接 .（即结束）」。 一个特殊符号干两件事，简化建模。

2.3 构建训练样本：所有相邻字符对

xs, ys = [], []
for w in words:
    chs = ['.'] + list(w) + ['.']
    for ch1, ch2 in zip(chs, chs[1:]):
        xs.append(stoi[ch1])  # 输入：前一个字符
        ys.append(stoi[ch2])  # 标签：后一个字符

import torch
xs = torch.tensor(xs)   # shape: (N,)
ys = torch.tensor(ys)   # shape: (N,)
# N ≈ 228,146 个 bigram 训练对
单词 "emma" 产生的训练对： ('.', 'e') → x=0, y=5 ('e', 'm') → x=5, y=13 ('m', 'm') → x=13, y=13 ('m', 'a') → x=13, y=1 ('a', '.') → x=1, y=0

3. 方法一：计数法（统计 Bigram 频次）

最朴素的想法：直接数数。"a 后面跟 b 出现了多少次" 这种统计量天然就是概率估计。

3.1 构建 27×27 频次矩阵 N

N = torch.zeros((27, 27), dtype=torch.int32)

for w in words:
    chs = ['.'] + list(w) + ['.']
    for ch1, ch2 in zip(chs, chs[1:]):
        ix1 = stoi[ch1]
        ix2 = stoi[ch2]
        N[ix1, ix2] += 1

N[i, j] = 字符 i 后面跟着字符 j 在训练集中出现的次数。 可视化（Karpathy 视频里这张 27×27 热力图很出名）能直接看到 "q→u"、 "."→ "a/e/.." 等明显的语言规律。

3.2 频次 → 概率（按行归一化）

P = N.float()
P = P / P.sum(dim=1, keepdim=True)   # 每一行加起来 = 1
# P[i, j] = P(下一个字符 = j | 当前字符 = i)
⚠️ 关于 keepdim=True（容易踩坑）
P.sum(dim=1) 形状是 (27,)， P.sum(dim=1, keepdim=True) 形状是 (27, 1)。 只有后者才能与 P 正确广播做行归一化。 Karpathy 在视频里专门花了几分钟讲 broadcasting，这正是昨天笔记里强调的「tensor shape 肌肉记忆」的重点。

3.3 采样（生成名字）

从 . 出发，根据当前字符所在行的概率分布采样下一个字符，遇到 . 停。

g = torch.Generator().manual_seed(2147483647)

for _ in range(10):
    out = []
    ix = 0  # 起点是 '.'
    while True:
        p = P[ix]                                          # 当前行的概率分布
        ix = torch.multinomial(p, num_samples=1,
                               replacement=True,
                               generator=g).item()
        if ix == 0:
            break
        out.append(itos[ix])
    print(''.join(out))
# 典型输出：mor, axx, minaymoryles, kondlaisah, anchshizarie, ...

结果有点像名字，但还很"乱" —— 这正是 Bigram 模型的天花板（只看 1 个字符的记忆实在太短）。

4. 评估模型：似然与 NLL Loss

"生成出来像不像名字"是主观感受，工程上需要一个数字化指标。 语言模型最经典的指标就是 训练集上的似然。

4.1 似然 (Likelihood)

训练集所有 bigram 对的联合概率：

$$ L = \prod_{i=1}^{N} P(y_i \mid x_i) $$

这个值会非常非常小（一堆 0.x 连乘），数值上会下溢出。所以取对数：

$$ \log L = \sum_{i=1}^{N} \log P(y_i \mid x_i) $$

4.2 负对数似然 (NLL) — 作为 Loss

我们希望 likelihood 越大越好 ⇒ log-likelihood 越大越好 ⇒ 负对数似然越小越好。 通常再除以 N 取平均，得到每个 bigram 的平均 NLL：

$$ \mathcal{L} = -\frac{1}{N} \sum_{i=1}^{N} \log P(y_i \mid x_i) $$

log_likelihood = 0.0
n = 0
for w in words:
    chs = ['.'] + list(w) + ['.']
    for ch1, ch2 in zip(chs, chs[1:]):
        ix1, ix2 = stoi[ch1], stoi[ch2]
        prob = P[ix1, ix2]
        log_likelihood += torch.log(prob)
        n += 1

nll = -log_likelihood / n
print(f'nll = {nll.item():.4f}')   # ≈ 2.4543
核心直觉：NLL 就是「平均每一步预测有多差」。 对于词表大小 27，如果模型完全瞎猜（均匀分布），NLL = ln(27) ≈ 3.30； Bigram 计数法做到 ≈ 2.45，说明确实学到东西了。 后面 MLP / Transformer 会把这个数字一路降到 2.1、1.9、1.7…

5. 模型平滑（防止 log(0))

如果某个 bigram 在训练集里从没出现过，比如 P("jq"→"z")=0， 那么 log(0) = -∞，NLL 就爆了。

解决方法：加法平滑（Laplace smoothing），把所有计数都 +k：

P = (N + 1).float()                  # 平滑系数 k=1
P = P / P.sum(dim=1, keepdim=True)

k 越大，分布越"平"（越接近均匀分布），相当于正则化。这一思想在后面神经网络里对应权重正则项 λ * (W**2).mean()。

6. 方法二：神经网络法（与计数法等价）

这一节是这一集的真正高潮。Karpathy 想让你看到： 计数法不是另一种方法，它本质上就是一个最简单的神经网络。 搞懂这个之后，把"前 1 个字符"换成"前 N 个字符"、把单层线性换成 MLP/Transformer， 你就理解了从 Bigram 到 GPT 的全部演进。

6.1 模型结构

输入 x (一个字符的索引, 0~26) │ ▼ one-hot 编码: (27,) ← F.one_hot(x, 27) │ ▼ 线性层: y = x @ W W shape: (27, 27), 无 bias │ ▼ logits (27,) 每个位置一个"原始分数" │ ▼ exp + 归一化 = softmax │ ▼ probs (27,) 概率分布，和 = 1

关键观察：参数矩阵 W 的形状是 (27, 27)， 和计数法的频次矩阵 N 形状一模一样！只是 W 不是从数据数出来的， 而是通过梯度下降学出来的。

6.2 前向传播代码

import torch
import torch.nn.functional as F

# 初始化参数
g = torch.Generator().manual_seed(2147483647)
W = torch.randn((27, 27), generator=g, requires_grad=True)

# === Forward Pass ===
xenc = F.one_hot(xs, num_classes=27).float()  # (N, 27)
logits = xenc @ W                              # (N, 27)  == log-counts
counts = logits.exp()                          # 等价于 N 矩阵
probs = counts / counts.sum(1, keepdim=True)   # softmax: (N, 27)
🔍 关键 insight（Karpathy 反复强调的）
  • xenc @ W 由于 xenc 是 one-hot， 其结果等价于「从 W 中取出第 x 行」。
  • 因此可以把 W[i, :] 理解为「字符 i 后面接各个字符的 log-counts（对数频次）」。
  • exp 把 log-counts 还原成 counts；除以 sum 就是按行归一化得到概率。
  • 整个网络就是一个把"频次矩阵 N"用"可学习的对数空间参数 W"重新参数化的过程。

6.3 损失函数：负对数似然

# 取出真实标签位置上的概率
loss = -probs[torch.arange(N), ys].log().mean()

等价于 F.cross_entropy(logits, ys)（数值上更稳）。 Karpathy 在视频里特意先手写一遍，让你彻底看清 "cross_entropy = softmax + log + 取标签位置 + 取负 + 求均值"。

6.4 反向传播 & 梯度下降

num_steps = 100
lr = 50  # bigram 这种小网络可以用很大的 lr

for k in range(num_steps):
    # ---- forward ----
    xenc = F.one_hot(xs, num_classes=27).float()
    logits = xenc @ W
    counts = logits.exp()
    probs = counts / counts.sum(1, keepdim=True)
    loss = -probs[torch.arange(len(xs)), ys].log().mean() \
           + 0.01 * (W**2).mean()           # L2 正则（= 平滑）

    # ---- backward ----
    W.grad = None
    loss.backward()

    # ---- update ----
    W.data += -lr * W.grad

    if k % 10 == 0:
        print(f'step {k:3d}  loss = {loss.item():.4f}')

训练几十步之后，loss 会收敛到 ≈ 2.47，几乎等于纯计数法的 2.4543 —— 这就是 "two methods produce the same model" 的实证。

那为什么我们还要绕一圈用神经网络？
因为计数法不可扩展：
  • 如果上下文是 2 个字符 → 矩阵变 27² × 27 = 19,683 行 → 可接受
  • 3 个字符 → 27³ × 27 = 531,441 行 → 已经很糟
  • 10 个字符 → 27¹⁰ × 27 ≈ 5×10¹⁵ 行 → 完全不可能
而神经网络的参数量可以独立于上下文长度（用 embedding + MLP/Transformer 压缩信息）， 这就是从 Bigram → MLP → Transformer 的根本动机。

7. 两种方法的对照

维度 计数法 神经网络法
核心数据结构频次矩阵 N (27×27)权重矩阵 W (27×27)
"训练"方式遍历一次数据，数数梯度下降迭代
得到概率N 按行归一化softmax(W[x])
正则化方式加法平滑 N+kL2: λ·mean(W²)
训练集 NLL≈ 2.4543≈ 2.4700
可扩展性❌ 上下文一长就爆炸✅ 参数量可解耦
对应的 ML 范式非参数统计参数化模型 + 优化

8. 为什么这一集是 Transformer 的起点

Karpathy 设计这一集的真正目的，是让你把后面所有复杂模型套到同一个心智模型上：

输入 token 序列 ──► [某种 architecture] ──► logits ──► softmax ──► probs ↑ ↑ Bigram: W NLL loss MLP : embedding + hidden + W RNN : embedding + 循环状态 + W Transformer: embedding + self-attention 堆叠 + W

整个 Zero to Hero 系列的脉络是：「Architecture 在变，目标/损失/采样/评估都不变」。 这正是工程师能稳稳跟上后面集数的关键 —— 因为「外壳」已经在这一集打牢了。

9. 给 AI Infra / Kafka 背景读者的视角

🛠 从 Infra 视角看这一集，关注什么
  1. Tensor shape 的肌肉记忆：xs (N,)、xenc (N, 27)、 W (27, 27)、logits (N, 27)、probs (N, 27)。 每一步都要能默写出 shape 变化 —— 这是后面读 vLLM、Megatron 源码的入场券。
  2. xenc @ W 在 one-hot 下退化成查表。 这就是 nn.Embedding 的本质 —— 不是"魔法"，就是"用整数索引代替 one-hot + matmul" 的访存优化。从 Infra 视角看，这是一次从 "稀疏 matmul" 到 "gather" 的算子重写。
  3. 正则项 = 平滑：0.01 * (W**2).mean() 在数学上等价于给 N 加常数。 ML 里的"两种世界观"在工程实现上其实是同一份代码的两种解释。
  4. 采样是顺序的、batch 1 的：每生成一个字符要 forward 一次。 到 GPT 推理时，这就是 decode 阶段 —— autoregressive、IO bound、KV Cache 的用武之地。 你现在写的这个while ix != 0: forward; sample 循环，本质就是 vLLM 的 decode loop 最简化版。
  5. vocab_size=27 很小，所以 softmax 没成本； 但在真实 LLM 中 vocab_size = 32k~150k，logits.exp()、按行求和、 cross_entropy 都会成为显存与算力大头，催生了诸如 Tied Embedding、 Fused CrossEntropy 等优化点。

10. 一份可直接运行的完整代码

下面是把上面所有内容串起来的最小可运行脚本。把 names.txt 放到同目录即可 （数据集来自 karpathy/makemore）。

"""
makemore Ep.1  —— Bigram 字符级语言模型
两种方法对照实现：计数法 vs 神经网络法
"""
import torch
import torch.nn.functional as F

# ============ 1. 读数据 ============
words = open('names.txt').read().splitlines()
print(f'#words = {len(words)},  e.g. {words[:5]}')

# ============ 2. 字符表 ============
chars = sorted(set(''.join(words)))
stoi = {s: i + 1 for i, s in enumerate(chars)}
stoi['.'] = 0
itos = {i: s for s, i in stoi.items()}
V = len(stoi)

# ============ 3. 构建训练样本 ============
xs, ys = [], []
for w in words:
    chs = ['.'] + list(w) + ['.']
    for a, b in zip(chs, chs[1:]):
        xs.append(stoi[a])
        ys.append(stoi[b])
xs = torch.tensor(xs)
ys = torch.tensor(ys)
N_total = xs.nelement()
print(f'#bigrams = {N_total}')

# ====================================================
# 方法一：计数法
# ====================================================
N = torch.zeros((V, V), dtype=torch.int32)
for x, y in zip(xs.tolist(), ys.tolist()):
    N[x, y] += 1

P = (N + 1).float()                       # 加 1 平滑
P /= P.sum(dim=1, keepdim=True)

# 评估 NLL
log_likelihood = 0.0
for x, y in zip(xs.tolist(), ys.tolist()):
    log_likelihood += torch.log(P[x, y])
nll = -log_likelihood / N_total
print(f'[Counting] NLL = {nll.item():.4f}')

# 采样
g = torch.Generator().manual_seed(2147483647)
print('[Counting] Samples:')
for _ in range(10):
    out, ix = [], 0
    while True:
        ix = torch.multinomial(P[ix], 1, replacement=True, generator=g).item()
        if ix == 0:
            break
        out.append(itos[ix])
    print('  ', ''.join(out))

# ====================================================
# 方法二：神经网络法
# ====================================================
g = torch.Generator().manual_seed(2147483647)
W = torch.randn((V, V), generator=g, requires_grad=True)

for step in range(200):
    xenc = F.one_hot(xs, num_classes=V).float()
    logits = xenc @ W
    # 等价于 F.cross_entropy(logits, ys)，这里手写为了暴露细节
    counts = logits.exp()
    probs = counts / counts.sum(1, keepdim=True)
    loss = -probs[torch.arange(N_total), ys].log().mean() \
           + 0.01 * (W ** 2).mean()

    W.grad = None
    loss.backward()
    W.data += -50 * W.grad

    if step % 20 == 0:
        print(f'  step {step:3d}  loss = {loss.item():.4f}')

print(f'[NeuralNet] NLL = {loss.item():.4f}')

# 神经网络采样
g = torch.Generator().manual_seed(2147483647)
print('[NeuralNet] Samples:')
for _ in range(10):
    out, ix = [], 0
    while True:
        xenc = F.one_hot(torch.tensor([ix]), num_classes=V).float()
        logits = xenc @ W
        probs = F.softmax(logits, dim=1)
        ix = torch.multinomial(probs, 1, generator=g).item()
        if ix == 0:
            break
        out.append(itos[ix])
    print('  ', ''.join(out))
预期输出：两种方法的 NLL 都在 2.45 ~ 2.47 之间， 生成的"名字"在两种方法下也非常相似（如 mor, axx, minaymoryles, ...）。 这就实证了"两种方法等价"。

📌 一句话总结

Bigram 模型用 27×27 的小矩阵讲清了语言模型的全部骨架：
    数据 → 转 id → 前向 → softmax → 概率 → NLL loss → 反传 → 更新 → 采样。
    后面 4 集（MLP / WaveNet / Transformer）唯一在变的是中间那个箱子， 其余的"管道"完全一致。
