1. 这一集到底在做什么

这一集复现的是 Bengio et al. 2003 的经典论文 "A Neural Probabilistic Language Model"， 也是神经语言模型的鼻祖。整个 GPT 家族都是从这篇论文的思路演化而来。

1.1 与 Ep.1 的差别

维度Ep.1 (Bigram)Ep.2 (MLP)
上下文长度1 个字符3 个字符（block_size=3）
字符表示one-hot (27 维)Embedding 向量 (2 维)
模型结构1 个矩阵 WEmbedding + 隐藏层 + 输出层
激活函数无（纯线性）tanh
训练方式全量梯度mini-batch SGD
数据划分不划分train/dev/test 80/10/10
调参几乎没有学习率搜索
dev loss≈ 2.45≈ 2.17 ~ 2.20

1.2 一句话目标

用前 3 个字符预测下一个字符，构建一个带 embedding + 隐藏层的小型神经语言模型。

2. 数据准备：构建 block_size=3 的训练样本

2.1 关键参数

block_size = 3   # 用前 3 个字符预测第 4 个
n_embd     = 2   # 每个字符的 embedding 维度（教学用，能可视化）
n_hidden   = 100 # 隐藏层神经元数
vocab_size = 27  # 26 字母 + 特殊 token '.'

2.2 滑动窗口构建样本

对每个名字，用一个长度为 3 的滑动窗口扫一遍：

def build_dataset(words):
    X, Y = [], []
    for w in words:
        context = [0] * block_size       # 初始上下文：[., ., .]
        for ch in w + '.':
            ix = stoi[ch]
            X.append(context)             # 当前上下文 → 输入
            Y.append(ix)                  # 下一个字符 → 标签
            context = context[1:] + [ix]  # 滑动窗口右移一格
    return torch.tensor(X), torch.tensor(Y)

2.3 具体例子

单词 "emma" 产生的训练样本： context → label (X 一行) (Y 一项) [. . .] → e [0, 0, 0] 5 [. . e] → m [0, 0, 5] 13 [. e m] → m [0, 5, 13] 13 [e m m] → a [5, 13, 13] 1 [m m a] → . [13, 13, 1] 0

最终的张量形状：

X.shape  # (N, 3)   N 个样本，每个是前 3 个字符的 id
Y.shape  # (N,)     N 个标签
# 32,033 个名字 → N ≈ 228,146 个训练样本

3. 模型结构：Embedding + 隐藏层 + 输出层

3.1 参数定义

g = torch.Generator().manual_seed(2147483647)

C  = torch.randn((vocab_size, n_embd), generator=g)             # (27, 2)   Embedding 表
W1 = torch.randn((block_size * n_embd, n_hidden), generator=g)  # (6, 100)
b1 = torch.randn(n_hidden, generator=g)                          # (100,)
W2 = torch.randn((n_hidden, vocab_size), generator=g)            # (100, 27)
b2 = torch.randn(vocab_size, generator=g)                        # (27,)

parameters = [C, W1, b1, W2, b2]
for p in parameters:
    p.requires_grad = True

print(sum(p.nelement() for p in parameters))   # 约 3,481 个参数

3.2 数据流（带 shape）

X (N, 3) ── gather (C[X]) ──► emb (N, 3, 2) │ view(-1, 6) ← 零拷贝展平 ▼ (N, 6) │ @ W1 + b1 ▼ (N, 100) │ tanh() ▼ h (N, 100) │ @ W2 + b2 ▼ logits (N, 27) │ F.cross_entropy(., Y) ▼ loss (标量)
必须能默写出来的 shape 链：
X (N,3) → emb (N,3,2) → (N,6) → h (N,100) → logits (N,27) → loss

4. 关键操作：从 (N,3,2) 到 (N,6) 的零拷贝展平

Karpathy 在视频里花了不少时间讲这个，因为它是第一次让你直面 PyTorch 的张量布局。

4.1 为什么必须展平

下一层是普通的 Linear：h = emb_cat @ W1 + b1，需要 2D 矩阵作输入。 但 emb = C[X] 是 3D 的 (N, 3, 2)，必须把"3 个字符 × 每个 2 维"拼成"每个样本 6 维"。

4.2 三种写法

写法复制内存通用性
torch.cat([emb[:,0,:], emb[:,1,:], emb[:,2,:]], dim=1) ✅ 复制 写死 block_size=3
torch.cat(torch.unbind(emb, 1), dim=1) ✅ 复制 通用但慢
emb.view(emb.shape[0], -1) ⭐ ❌ 零拷贝 最通用，最快
⚙️ 为什么 view 是零拷贝
PyTorch 张量在底层是一段连续的一维内存，shape 和 stride 只是"读取这段内存的协议"。 view 只改协议，不动数据。当原张量是 contiguous 的，view 就能成功， 且瞬时完成（O(1) 时间，0 字节复制）。这是 GPU 上张量操作之所以快的核心机制之一。

5. tanh 激活：神经网络非线性的来源

没有 tanh 会怎样？两个 Linear 层叠在一起：

h     = X @ W1 + b1
out   = h @ W2 + b2
    = X @ (W1 @ W2) + (b1 @ W2 + b2)
    = X @ W' + b'     ← 等价于一个单层！

→ 不管堆多少层，没有非线性激活，最终都退化为一层线性变换。

5.1 tanh 的形状与性质

tanh(x) = (e^x - e^(-x)) / (e^x + e^(-x)) +1 ┤━━━━━━━╱───────── ← 顶部饱和 │ ╱ │ ╱ ← 中间近似线性 │ ╱ 0 ─┼───────────── │ ╱ │ -1 ┤─━━━━━━━━━━━━━━ ← 底部饱和 -3 -1 0 1 3 tanh(0) = 0 tanh(3) ≈ +0.995 tanh(-3) ≈ -0.995

5.2 在模型里的位置

h_pre = emb_cat @ W1 + b1     # (N, 100)  线性变换后的"激活前"
h     = torch.tanh(h_pre)     # (N, 100)  挤压到 [-1, 1]，引入非线性
💡 "激活"是什么意思
这个词来自 1940 年代生物神经元建模 —— 生物神经元"放电"叫 "fire / activate"。 在人工神经网络里，激活函数就是"决定一个神经元此刻输出多大（多兴奋）的函数"， 它的输出就叫激活值（activation）。
虽然现代 NN 已经不像生物了，但这个词留了下来。

6. Cross-Entropy Loss：fused softmax + log + nll

Ep.1 里我们是手写的：

# 手写版（Ep.1）
counts = logits.exp()
probs = counts / counts.sum(1, keepdim=True)
loss = -probs[torch.arange(N), Y].log().mean()

Ep.2 改成一行：

loss = F.cross_entropy(logits, Y)

6.1 为什么用 F.cross_entropy 而不是手写

  1. 数值稳定：手写版 logits.exp() 在 logits 很大时（比如 +50）会溢出成 inf。 F.cross_entropy 内部用 LogSumExp trick（减去 max 后再 exp）避免上溢， 并且把 log(softmax(.)) 合并计算避免下溢。
  2. 性能更好：把 softmax + log + gather + mean 融合成一个 kernel，省一次显存读写。 这就是 GPU 上"算子融合（kernel fusion）"的雏形。
  3. 反向传播更稳：合并算梯度时数学公式化简后只剩 probs - one_hot(Y)，简单又稳。
🛠 Infra 视角
F.cross_entropy 在 LLM 训练里是个老熟人 —— 在 vocab_size=50k+ 的真实场景下， 这一个 kernel 占的显存和算力都不小。Megatron 和 vLLM 都有专门优化的 fused cross-entropy 实现。

7. Mini-batch 训练循环

Ep.1 是全量训练（一次看 N≈228k 个样本）。Ep.2 改成每次只看 32 个：

batch_size = 32

for step in range(200000):
    # === 抽 batch ===
    ix = torch.randint(0, Xtr.shape[0], (batch_size,))  # 32 个随机样本编号
    
    # === forward ===
    emb = C[Xtr[ix]]                            # (32, 3, 2)
    h = torch.tanh(emb.view(-1, 6) @ W1 + b1)   # (32, 100)
    logits = h @ W2 + b2                        # (32, 27)
    loss = F.cross_entropy(logits, Ytr[ix])
    
    # === backward ===
    for p in parameters:
        p.grad = None
    loss.backward()
    
    # === update ===
    lr = 0.1 if step < 100000 else 0.01
    for p in parameters:
        p.data += -lr * p.grad

7.1 Mini-batch 的 3 个好处

  1. 显存可控：真实数据集可能几亿样本，全量算不了。Mini-batch 把"一次大计算"切成"很多次小计算"。
  2. 更新更快：32 个样本算一次近似梯度就能更新，比全量快得多。虽然梯度有噪声，但更新次数多，整体收敛更快。
  3. 正则化作用：噪声梯度反而能帮模型跳出局部最优、避免过拟合。

8. 学习率搜索（lr finder）

Karpathy 演示了一个简单又实用的技巧：在对数空间扫描 lr，画图找拐点。

lre = torch.linspace(-3, 0, 1000)   # [-3, 0]，1000 个点
lrs = 10 ** lre                      # 对应 [0.001, 1.0]，对数均匀

losses = []
for step in range(1000):
    # ... forward + backward ...
    lr = lrs[step]                   # 当前步用第 step 个学习率
    for p in parameters:
        p.data += -lr * p.grad
    losses.append(loss.item())

import matplotlib.pyplot as plt
plt.plot(lre, losses)                # x 轴是 log10(lr)，y 轴是 loss

8.1 怎么读这张图

loss │ │ ╲ ╱ │ ╲ ╱ │ ╲ ─────────── ╲╲ ╱ ← 这一段最平稳，是合适的 lr 区间 │ ╲ ╱ │ ╲╲╲╲╱ ← 太大，loss 反弹 │ └────────────────────────── log10(lr) -3 -2 -1 0 0.001 0.01 0.1 1.0

选"loss 平稳下降到拐点"附近的 lr 作为正式训练用的值。这一集里 Karpathy 选了 lr = 0.1， 后期降到 0.01 做"learning rate decay"。

9. Train / Dev / Test 划分

第一次出现"科学评估"。把全部数据按 80% / 10% / 10% 分成三份：

import random
random.seed(42)
random.shuffle(words)

n1 = int(0.8 * len(words))
n2 = int(0.9 * len(words))

Xtr,  Ytr  = build_dataset(words[:n1])       # 训练集
Xdev, Ydev = build_dataset(words[n1:n2])     # 验证集（调参用）
Xte,  Yte  = build_dataset(words[n2:])       # 测试集（最终评估，只看一次）

9.1 三者的角色

数据集用途看几次
train计算梯度更新参数无数次
dev (validation)调超参（lr、隐藏层大小、epoch 数等）多次
test报告最终性能只看 1 次（论文/模型卡用）
⚠️ 经典错误
用 test set 调参 → test loss 会越调越低，但这是假象，你只是把超参过拟合到了 test set。 正确做法：dev 调到尽兴，最后跑一次 test 报数字，不许再调。

10. Embedding 可视化与模型容量

10.1 看看 embedding 学到了什么

因为 n_embd=2，可以直接在二维平面上画出 27 个字符的位置：

plt.scatter(C[:, 0].data, C[:, 1].data)
for i in range(27):
    plt.text(C[i, 0].item(), C[i, 1].item(), itos[i])

训练完你会看到：

  • 元音 a/e/i/o/u 聚成一团
  • 辅音聚成另一团
  • 特殊 token '.' 单独占据某个角落

这就是 embedding 的魅力 —— 没人告诉模型什么是元音，它从统计中自己学到了。

10.2 提升模型容量的几种方法

这一集的最终 dev loss ≈ 2.17，比 Bigram 的 2.45 大幅下降。还能怎么继续降？

方法怎么改预期效果
加大 embedding 维度n_embd: 2 → 10loss 降到 ~2.13
加大隐藏层n_hidden: 100 → 300loss 再降一点
加长上下文block_size: 3 → 5/8需要更多隐藏层容量配合
更多训练步数更长训练边际递减

11. Infra 视角：这一集埋下的工程伏笔

🛠 给做基础设施的人看的几个真伏笔
  1. Embedding = gather kernel
    C[X] 在 GPU 上是 gather，不是 matmul。这是 nn.Embedding 的底层实现， 也是后面 LLM 训练里那张 embedding 表占大量显存（LLaMA-7B 256MB）的根源。
  2. view 的零拷贝 = shape/stride 协议
    张量在底层是一维内存 + (shape, stride) 元数据。view、transpose、 permute 只改元数据。这跟 OS 虚拟内存 / 数据库列视图是同构的"逻辑视图 vs 物理布局"思想。 不连续的张量上用 view 会报错 —— 这个报错本身就是个有用的信号：提醒你这里可能有数据复制开销。
  3. F.cross_entropy = 算子融合的雏形
    把 softmax + log + gather + mean 合并成一个 kernel，避免中间张量物化。 到 LLM 训练时这变成显存大头之一，Megatron 有专门的 fused vocab parallel cross entropy。
  4. Mini-batch ≠ batch_size 不影响模型
    batch_size 是个工程参数（决定显存占用、吞吐、梯度噪声），跟模型本身无关。 在 vLLM 里，"batching" 是另一个维度：continuous batching 把多个推理请求动态合并， 是 LLM 推理吞吐的关键。底层都是 GPU 喜欢"批量"这个事实。
  5. Train/Dev/Test 在大模型时代怎么变
    现代 LLM 的"评估"已经不只是 dev loss 了 —— 还要 MMLU、HumanEval、GSM8K 等下游任务。 但 dev loss 仍然是训练时盯吃饭的指标（loss spike 一般意味着不稳定/数值问题）。

12. 一份可直接运行的完整代码

把视频里所有关键步骤串起来的最小可运行版本。把 names.txt 放到同目录即可。

"""
makemore Ep.2 —— MLP 字符级语言模型
按 Bengio 2003 论文复现：embedding + 隐藏层 + 输出层
"""
import torch
import torch.nn.functional as F
import random

# ============ 1. 读数据 + 构建字符表 ============
words = open('names.txt').read().splitlines()
chars = sorted(set(''.join(words)))
stoi = {s: i + 1 for i, s in enumerate(chars)}
stoi['.'] = 0
itos = {i: s for s, i in stoi.items()}
V = len(stoi)

# ============ 2. 构建训练样本（block_size=3） ============
block_size = 3
def build_dataset(ws):
    X, Y = [], []
    for w in ws:
        ctx = [0] * block_size
        for ch in w + '.':
            ix = stoi[ch]
            X.append(ctx)
            Y.append(ix)
            ctx = ctx[1:] + [ix]
    return torch.tensor(X), torch.tensor(Y)

random.seed(42)
random.shuffle(words)
n1, n2 = int(0.8 * len(words)), int(0.9 * len(words))
Xtr,  Ytr  = build_dataset(words[:n1])
Xdev, Ydev = build_dataset(words[n1:n2])
Xte,  Yte  = build_dataset(words[n2:])
print(f'train: {Xtr.shape}, dev: {Xdev.shape}, test: {Xte.shape}')

# ============ 3. 模型参数 ============
n_embd, n_hidden = 10, 200   # 实战推荐值（Karpathy 视频后期使用）
g = torch.Generator().manual_seed(2147483647)

C  = torch.randn((V, n_embd), generator=g)
W1 = torch.randn((block_size * n_embd, n_hidden), generator=g)
b1 = torch.randn(n_hidden, generator=g)
W2 = torch.randn((n_hidden, V), generator=g)
b2 = torch.randn(V, generator=g)
parameters = [C, W1, b1, W2, b2]
for p in parameters:
    p.requires_grad = True
print(f'#params = {sum(p.nelement() for p in parameters)}')

# ============ 4. 训练循环 ============
batch_size = 32
for step in range(200000):
    ix = torch.randint(0, Xtr.shape[0], (batch_size,))
    
    emb = C[Xtr[ix]]                                   # (32, 3, 10)
    h = torch.tanh(emb.view(-1, block_size * n_embd) @ W1 + b1)  # (32, 200)
    logits = h @ W2 + b2                               # (32, 27)
    loss = F.cross_entropy(logits, Ytr[ix])
    
    for p in parameters:
        p.grad = None
    loss.backward()
    
    lr = 0.1 if step < 100000 else 0.01
    for p in parameters:
        p.data += -lr * p.grad
    
    if step % 10000 == 0:
        print(f'step {step:6d}  loss = {loss.item():.4f}')

# ============ 5. 评估 dev loss ============
@torch.no_grad()
def split_loss(split):
    X, Y = {'train': (Xtr, Ytr), 'dev': (Xdev, Ydev), 'test': (Xte, Yte)}[split]
    emb = C[X]
    h = torch.tanh(emb.view(-1, block_size * n_embd) @ W1 + b1)
    logits = h @ W2 + b2
    loss = F.cross_entropy(logits, Y)
    print(f'{split} loss = {loss.item():.4f}')

split_loss('train')
split_loss('dev')

# ============ 6. 采样生成新名字 ============
g = torch.Generator().manual_seed(2147483647 + 10)
for _ in range(20):
    out, ctx = [], [0] * block_size
    while True:
        emb = C[torch.tensor([ctx])]
        h = torch.tanh(emb.view(1, -1) @ W1 + b1)
        logits = h @ W2 + b2
        probs = F.softmax(logits, dim=1)
        ix = torch.multinomial(probs, 1, generator=g).item()
        ctx = ctx[1:] + [ix]
        if ix == 0:
            break
        out.append(itos[ix])
    print(''.join(out))
预期结果：
  • train loss ≈ 2.12, dev loss ≈ 2.17（n_embd=10, n_hidden=200）
  • 生成样本如 carmah, amelle, khimrix, jenne, taparelle —— 明显比 Ep.1 的 axx, minaymoryles 更像真名字

13. ⭐ 学习过程中遇到的 PyTorch 概念答疑（高频卡点）

这一集的"理论"不难，难的是 PyTorch 张量操作的肌肉记忆。下面是我在学这一集时实际卡住的所有问题，按出现顺序整理。如果你也在同样的地方卡住，看这里就够了。

Q1：emb = C[X] 为什么 emb 的 shape 是 (N, 3, 2)，不是 2 维？

核心规则（PyTorch fancy indexing）：

用 X 里的每个整数去 C 里取对应那一行，X 的形状原样保留，并在末尾追加被取出来那一行的形状。

套到这里：

X.shape    = (N, 3)        ← X 里每个元素是 0~26 的索引
C[i].shape = (2,)          ← 从 C 取一行得到 2 维向量

C[X].shape = (N, 3) + (2,) = (N, 3, 2)

语义：N 个样本，每个样本由 3 个字符组成，每个字符被表示成 2 维 embedding 向量。

Q2：emb[:, 0, :] 是什么含义？

PyTorch 索引规则：逗号分隔每个维度的取法，: 表示"这个维度全要"，整数表示"只取这个位置，并把这个维度去掉"。

所以 emb[:, 0, :] = "所有样本（dim=0 全要），每个样本只取第 0 个字符（dim=1 取 0，这一维消失），该字符的 embedding 全要（dim=2 全要）"。

emb         shape (N, 3, 2)
emb[:,0,:]  shape (N, 2)    ← dim=1 被切掉了

三兄弟 emb[:,0,:]、emb[:,1,:]、emb[:,2,:] 拼接起来就是 (N, 6)，效果和 emb.view(N, -1) 一样。

Q3：emb.view(emb.shape[0], -1) 怎么理解？

三部分拆开：

  • emb.shape[0] = N，即第 0 维的大小（写 emb.shape[0] 不写死数字，是为了不依赖具体 batch size）
  • -1 = "占位符，让 PyTorch 自动算"，约束是总元素数对得上：N×3×2 = 6N，已指定第 0 维=N，所以第 1 维=6
  • .view(...) = "换一副形状的眼镜看同一块内存"，不复制数据

整行翻译："把 emb 重新解读为：第 0 维是样本数（保留），剩下所有维度铺平拼到第 1 维。"

验证：

v = emb.view(emb.shape[0], -1)
emb.data_ptr() == v.data_ptr()    # True，同一块内存
v[0, 0] = 999
print(emb[0, 0, 0])               # 999，改 view 也改了原张量
Q4：stride 是什么？

一句话定义：stride[i] 表示在第 i 维上前进 1 步，需要在底层一维内存里跨过多少个元素。

x = torch.arange(12).reshape(3, 4)
x.shape    # (3, 4)
x.stride() # (4, 1)

含义：

  • dim=0 stride=4：行索引 +1 → 内存跳 4 个元素（一整行长度）
  • dim=1 stride=1：列索引 +1 → 内存跳 1 个元素（相邻列）

寻址公式：x[i, j] 实际访问内存位置 = i * stride[0] + j * stride[1]

关键威力：改 stride 就能"零拷贝换视图"：

  • view：改 shape + 改 stride，不动数据
  • transpose：只交换 stride，不动数据（所以 transpose 后张量不连续）
  • contiguous()：真复制一份让 stride 重新连续

类比："shape + stride" 之于张量，等于 "页表 + 偏移" 之于虚拟内存。

Q5：embedding 到底是什么？看起来就是个多维向量？

是的，形式上就是个向量。但有两层：

形式层：C = torch.randn((27, 2))，每个字符对应一行 2 维向量。C[1] 就是字符 'a' 的 embedding。

语义层：embedding 替代了 one-hot，解决了三个问题：

  • 维度问题：one-hot 是词表大小（27 或 50000），稀疏浪费；embedding 维度可以自由选
  • 距离问题：one-hot 下任意两个字符距离相等；embedding 可以让"相似字符靠近"
  • 不可微：one-hot 是固定的；embedding 是可学习参数，梯度会推它去自动学结构

训练后：a/e/i/o/u 这种相似字符会自发聚成一团，而没人告诉过模型它们是元音。

一句话：embedding = 一张"字符 → 稠密低维向量"的可学习查找表，把离散符号嵌入到连续空间里。

Q6：embedding 长度为什么不可以是 1？

可以跑，但用了等于没用。原因有三：

  1. 一维空间表达力崩塌：一维直线只能表达"线性序"（谁在左谁在右），无法让 'a' 同时靠近多个不同的字符
  2. 信息瓶颈：3 个字符的 embedding 一共只剩 3 个数，后面 100 维隐藏层全从这 3 个数派生
  3. 实测退化到 Bigram：n_embd=1 时 dev loss ≈ 2.45，跟纯计数法持平

Karpathy 选 2 是为了可视化（能画在屏幕上）；实战至少 ≥ 10，LLaMA-7B 用 4096。

Q7：exp 和 log 在深度学习里干嘛用？

两个核心用法：

用法 1：exp 用在 softmax 里 —— 把任意分数变成概率

counts = logits.exp()         # 把任意实数变成正数
probs = counts / counts.sum() # 归一化，和为 1

用 exp 因为它永远输出正数 + 可微 + 保持相对大小。

用法 2：log 用在 loss 里 —— 把乘法变加法

似然 = N 个概率连乘，会下溢出。log 把乘法变加法，避免溢出：log(p1 * p2) = log(p1) + log(p2)

NLL Loss = -log(probs[正确标签])，信息论意义是"模型对真实标签的惊讶程度"。模型越自信，loss 越小。

注意数值稳定：exp 在大数下会溢出（exp(100) ≈ 10⁴³），所以标准做法是先减最大值。F.cross_entropy 内部已经处理好了，实战中直接用它，别手写。

Q8：tanh 是什么？

双曲正切函数：把任意实数挤压到 (-1, +1) 的 S 形曲线。

  • tanh(0) = 0，tanh(3) ≈ +0.995，tanh(-3) ≈ -0.995
  • 中间近似线性，两端"饱和"

它存在的唯一使命：在线性层之间引入非线性。没有它，再深的网络都等于一层线性变换。

缺点：饱和区梯度接近 0 → 梯度消失。所以现代 LLM 几乎不用 tanh 了，换成 ReLU / GELU / SwiGLU。但思想完全一样。

Q9："激活函数"的"激活"两个字是什么含义？

来自 1940 年代的生物神经元建模。生物神经元"放电"叫 fire / activate（激活）。

人工神经网络借用了这个词："激活函数" = 决定一个神经元此刻输出多大（多兴奋）的函数，它的输出叫"激活值（activation）"。

名字描述的是"它的输出代表什么（兴奋程度）"，不是描述"它对输入做了什么"（实际上是挤压）。

在代码里，"activations" 经常指中间层的输出 h，比如"激活内存爆了"指的是中间层输出占的显存。

Q10：Wx + b 这种函数叫什么"函数"？

三个层次的叫法：

  • 数学严谨：仿射变换（affine transformation）
  • 工程约定：线性层（linear layer）—— PyTorch 的 nn.Linear
  • 结构视角：全连接层 / FC layer / Dense layer

严格数学意义上"线性"要求过原点（即 b=0），但深度学习里宽松地把 Wx + b 都叫"线性"，这是约定俗成。

论文里看到 affine / linear / FC / Dense / FFN，都是同一个东西。

Q11：X[ix] 是什么？为什么 shape 变成 (32, 3)？

ix 是 32 个整数（mini-batch 编号），X[ix] 表示用 ix 里的每个整数当行号，从 X 抽出对应行组成新矩阵。

用同样的 fancy indexing 公式：

X.shape          = (N, 3)        被索引的是 dim=0
X[i] 的 shape    = (3,)
ix.shape         = (32,)

X[ix].shape = ix.shape + (3,) = (32, 3)

语义："从训练集随机抽 32 个样本组成一个 batch"。允许重复（torch.randint 是有放回抽样）。

Q12：C[X[ix]] 嵌套索引怎么理解？

按"从内到外"拆开：

  1. 先算 X[ix] → (32, 3)，得到 batch 里所有字符的 id
  2. 再算 C[X[ix]] → (32, 3, 2)，把每个 id 翻译成 embedding
ix.shape       = (32,)
X[ix].shape    = (32, 3)        ← 第一次 gather
C[X[ix]].shape = (32, 3, 2)     ← 第二次 gather

一行做了两件事：mini-batch 抽样 + embedding lookup。等价于：

Xb = X[ix]      # (32, 3)
emb = C[Xb]     # (32, 3, 2)
📌 这 12 个 Q 总结成一句话：
PyTorch 张量操作不难，但需要记住三个核心规则：
① fancy indexing 的 shape 推导：结果 = 索引 shape + 被索引那维之后保留的 shape
② view / transpose / contiguous 的零拷贝原理：shape + stride 是内存解读协议，改协议不动数据
③ exp / log / softmax / cross_entropy 的数值稳定性：实战别手写，用 F.cross_entropy
